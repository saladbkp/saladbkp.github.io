FTP and NFS Overview

What is NFS?

  • FTP (File Transfer Protocol): Involves uploading and downloading files, while NFS (Network File System) enables remote file access, resembling local file access from the clientâ€™s perspective. NFS operates as a form of Remote Procedure Call (RPC).

Stateless vs Stateful Systems

  • Stateful System: Server maintains file state information, providing a file descriptor to the client. The server handles state information, offering a more convenient but complex recovery process in case of crashes.
  • Stateless System: Client manages file state details, including file openness and offset, requiring self-contained requests. While simpler in normal operation, crash recovery becomes challenging due to potential data loss or inconsistency.

Challenges of Stateful and Stateless Modes

  • Stateful Challenges: Recovery complexity arises if the server crashes after providing a file descriptor, leading to potential loss of state information.
  • Stateless Challenges: In case of server crashes, the client continuously retries requests, making it challenging to distinguish between a crash and slow server response. Write operations in a stateless mode may mislead the client if data hasnâ€™t been written to disk (write-through).

NFS Workflow (Ubuntu YTB)

NFS Server

An NFS server operates as a client-server protocol and must meet the following requirements:

  • Has NFS server software installed.
  • Possesses at least one network connection for sharing NFS resources.
  • Is configured to accept and respond to NFS requests over the network connection.

NFS Client

An NFS client, interacting with the NFS server, must meet the following criteria:

  • Has NFS client software installed.
  • Maintains network connectivity to an NFS server.
  • Is authorized to access resources on the NFS server.
  • Is configured to send and receive NFS requests over the network connection.
