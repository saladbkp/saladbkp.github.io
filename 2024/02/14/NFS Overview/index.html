FTP and NFS Overview

What is NFS?

  • FTP (File Transfer Protocol): Involves uploading and downloading files, while NFS (Network File System) enables remote file access, resembling local file access from the client’s perspective. NFS operates as a form of Remote Procedure Call (RPC).

Stateless vs Stateful Systems

  • Stateful System: Server maintains file state information, providing a file descriptor to the client. The server handles state information, offering a more convenient but complex recovery process in case of crashes.
  • Stateless System: Client manages file state details, including file openness and offset, requiring self-contained requests. While simpler in normal operation, crash recovery becomes challenging due to potential data loss or inconsistency.

人话: stateful vs stateless

在stateful模式下，服务器维护文件状态信息，通过文件描述符传递给客户端，但在面对崩溃时，复杂性增加。

相反，在stateless模式下，客户端自己管理文件状态信息，包含所有必要信息在请求中，但在崩溃时可能导致请求重发和无法区分崩溃和缓慢响应的问题。

此外，写操作的一致性问题，强调了服务器在写入磁盘后才能回复客户端的重要性。

Challenges of Stateful and Stateless Modes

  • Stateful Challenges: Recovery complexity arises if the server crashes after providing a file descriptor, leading to potential loss of state information.
  • Stateless Challenges: In case of server crashes, the client continuously retries requests, making it challenging to distinguish between a crash and slow server response. Write operations in a stateless mode may mislead the client if data hasn’t been written to disk (write-through).

Version Evolution (v1-v4)

  • v1: Introduction of FTP and NFS.
  • v2: Emergence of stateful and stateless distinctions in system design.
  • v3: Highlighting challenges in crash recovery for both stateful and stateless modes.
  • v4: Recognizing nuances细微区别 in write operations and emphasizing the need for data consistency, particularly in the context of server crashes.

人话：2000年，NFS发展到了第4版（NFSv4），它转向了stateful（有状态）范式。解决了stateless（无状态）模式存在的问题 [Crash Recovery Complexity, Continuous Request Retry, Consistency in Write Operations]。Stateful模式不仅解决了已有的问题[Complexity of Crash Recovery, Close Operation]，还赋予了客户端close()操作新的含义：通知服务器丢弃关于该文件的状态信息。此外，NFSv4在多个方面进行了优化。

NFS Workflow (Ubuntu YTB)

NFS Server

An NFS server operates as a client-server protocol and must meet the following requirements:

  • Has NFS server software installed.
  • Possesses at least one network connection for sharing NFS resources.
  • Is configured to accept and respond to NFS requests over the network connection.

NFS Client

An NFS client, interacting with the NFS server, must meet the following criteria:

  • Has NFS client software installed.
  • Maintains network connectivity to an NFS server.
  • Is authorized to access resources on the NFS server.
  • Is configured to send and receive NFS requests over the network connection.
