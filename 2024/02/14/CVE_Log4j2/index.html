CVE Study 1 - Log4j2

Only IF

Explanation

The Log4j2 vulnerability (CVE-2021-44228) allows for remote code execution (RCE) through a feature called JNDI (Java Naming and Directory Interface) lookup. This feature is used by Log4j2 to perform dynamic variable substitution in its configuration.

人话：log4j2 的lookup output logging msg 让 developer access host 资源

When Log4j2 processes a log message containing a JNDI lookup, it tries to resolve the JNDI reference, which can be a URL pointing to a remote server. If the remote server is controlled by an attacker, they can craft a malicious JNDI reference that, when resolved by Log4j2, triggers the remote execution of arbitrary code on the vulnerable system.

jndi -> 请求普通数据

The RMI (Remote Method Invocation) communication is another aspect of this vulnerability. In some configurations, Log4j2 uses RMI to communicate with the remote JNDI server. By exploiting the JNDI lookup vulnerability, an attacker can control the RMI server’s response, leading to remote code execution on the target system.
