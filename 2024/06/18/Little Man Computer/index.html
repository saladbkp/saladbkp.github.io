Little Man Computer

Created2024-06-18|Updated2024-10-19|Study
|Word count:408|Reading time:2min|Post View:

Definition

Simulator that models a simple computer using von Neumann architecture

Programmed in assembly code + translates into machine code + load into RAM

Components

  • Program counter: store address of the next instruction in RAM
  • Instruction register: store OPP CODE of the current instruction
  • Address register: store address part of the current instruction
  • Accumulator: store result of any calculation

Process

1.Check the Program Counter to find the RAM address with the program instruction to be fetched.

2.Fetch the INSTRUCTION from that RAM address

3.Increment the Program Counter (so that it contains the RAM address of the next instruction).

4.Decode the instruction (this might include finding the RAM address for the data it will work on).

5.If necessary, fetch the DATA from the RAM address found in the previous step

6.Execute the instruction.

7.Repeat the cycle or halt

LMC Instruction Set

1st digit = instruction call

2nd and 3rd digit = mailbox address

Example: 255 = store into mailbox 25

Type of Instructions

Opcode Instruction
1 ADD
2 SUB
3 STORE
5 LOAD
6 Branch unconditionally (jmp)
7 Branch on zero
8 Branch on positive
0 Halt

902 Output
