Basic of Computer Theory

Created2024-06-18|Updated2024-06-18|Study
|Post View:

CPU

The function of CPU

Fetch instructions: from memory

Interpret instructions: decode them to determine action

Fetch data: for instruction execution

Process data: perform arithmetic operation

Write data: results are stored back in memory

CPU important part

Control Unit (fetch and decode instruction)

ALU (preform arithmetic +-/ logical and or not operations)

Registers (store temporary data)

Register

What is a register?

Small storage locations within the CPU used for a particular purpose

Manipulated directly by Control Unit

Wired for specific function

Size in bits or bytes (not MB like memory)

Can hold data, address, instruction

Why register

CPU must have some working space for temporary storage

CPU can’t directly perform arithmetic in memory

Store status of CPU and currently executing program

Register Operations

Storing value from other locations

Addition and subtraction

Shifting or rotating data

Testing contents for condition like zero or positive

Type Register Organization - Processor

User Visible Register UVR

  • Visible to programmer
  • May be general purpose or have a special use

User Invisible Register UIR

  • Used solely by CPU and special O/S functions
  • Used to control the operation of CPU
  • Special purpose or control register

UVR - General purpose register

Functionality: GPRs can hold any data type (integer, floating-point, address) and are used for a variety of operations, including arithmetic, logic, and data transfer operations.

Speed: Hold intermediate results or data values like loop counter

Quantity: Early computers had only one general purpose register which is accumulator. Typically, several dozen in current CPU and varies between different CPU architectures.

Versatility: GPRs are not limited to specific functions and can be used for multiple purposes within a program.

Performance: access to GPRs is much faster compared to accessing data in main memory, making them essential for efficient program execution.

UIV - Program Counter (PC)

    • Description: The PC, also known as the Instruction Pointer (IP) in some architectures, holds the memory address of the next instruction to be executed.
      • Function: It is automatically incremented after fetching an instruction and can be modified as a result of branch instructions, jumps, or procedure calls, guiding the control flow of programs.

UIV - Instruction Register (IR)

    • Description: The IR holds the currently executing instruction fetched from memory.
      • Function: It decodes and manages the execution of instructions, serving as a conduit through which the CPU understands and processes instructions sequentially.

UIV - Memory Address Register (MAR)

    • Description: The MAR holds the address in memory where the CPU either reads data from or writes data to. Contain address in memory to find or place data.
      • Function: It works closely with the Memory Data Register (MDR) during the fetch-decode-execute cycle, serving as the pointer to the specific memory location involved in an operation.

UIV - Memory Data Register (MDR)

    • Description: Also known as the Memory Buffer Register (MBR), the MDR temporarily holds data that is being transferred to or from memory. Contain actual data to be placed or retrieved in location given MAR.
      • Function: It acts as a buffer between the CPU and main memory, holding data that has been read from memory or data that is to be written to memory.

The MAR & MDR are used for data exchange between memory and the CPU

UIV - Status Register (SR)

    • Description: The Status Register, also known as the Flags Register or Program Status Word (PSW), contains individual bits that are set or cleared based on the outcomes of arithmetic and logical operations. Contain status information from program execution often error condition.
      • Function: Common flags include the Zero Flag (ZF), Carry Flag (CF), Half-Carry Flag (HC), Sign Flag (SF), Parity Flag (P), and Overflow Flag (OF). These flags are used for decision-making in conditional instructions.

UVR - Condition code register

to indicate the status of the CPU or the outcome of various operations

Exercise

i) Perform the operation (Addition) and find the result

Given two 8-bit operands:

  • Operand 1: 24 (in decimal)
  • Operand 2: 18 (in decimal)

Performing the addition: 24+18=4224 + 18 = 4224+18=42

ii) Find out the values of the binary Flags: S, Z, HC, C, P, V

Let’s analyse the flags based on the result of the addition (42 in decimal, or 2A in hexadecimal):

  • S (Sign flag): This flag is set if the result is negative (i.e., the most significant bit of the result is 1 for an 8-bit signed integer). For the result 42 (00101010 in binary), the most significant bit (bit 7) is 0.
    • S = 0 (The result is not negative)
  • Z (Zero flag): This flag is set if the result is zero. The result is 42.
    • Z = 0 (The result is not zero)
  • HC (Half-carry flag): This flag is set if there is a carry from bit 3 to bit 4 during the addition of the lower nibble.
    • Binary of 24 (operand 1): 00011000
    • Binary of 18 (operand 2): 00010010

1000 (8) + 0010 (2) = 1010 (10)

    • Adding lower nibbles (8 + 2): 0010 (in binary), there is NO carry from bit 3 to bit 4.
      • HC = 0 (There is no half-carry)
  • C (Carry flag): This flag is set if there is a carry out of the most significant bit (bit 7) during the addition. Since 24 + 18 = 42 is within the 8-bit range (0-255), there is no carry out.
    • C = 0 (There is no carry out)
  • P (Parity flag): This flag is set if the number of set bits (1s) in the result is even. Binary of 42 (00101010) has 3 set bits.
    • P = 0 (Parity is odd)
  • V (Overflow flag): This flag is set if there is an overflow, i.e., the result is out of the range of the signed 8-bit integer (-128 to 127). Since 24 + 18 = 42 is within the range, there is no overflow.
    • V = 0 (No overflow)

Machine cycle

The control unit performs its job by continually repeating an algorithm that guides it through a three-step process known as the machine cycle.

Fetch: Fetch the next instruction from memory that is stored at the address indicated by program counter (PC). Store the instructions in the Instruction Register (IR). PC is increased by 2 (each instruction has two bytes long).

Decode: Break the operand field into components based on the opcode.

Execute: Activate the appropriate circuit to perform the requested task.

Example load

  • Memory Size: 32 locations (addresses range from 0 to 31).
  • Instruction Format: Eight bits (1 byte).
  • Load Instruction Opcode: 010 (binary).
  • Value in Memory Location 15: 10 (decimal) which is 00001010 in binary.
  • Program Counter (PC): Currently at address 5, about to increment to 6.
  • Instruction in Memory Location 6: 01001111.
  • 010 (load) 1111 (15)

Register transfer language (RTL)

Format of RTL Statement:

  • Register source ————-> Register destination
  • Example: PC -> MAR
  • Use in fetch and execute phase
LOAD STORE ADD
PC -> MAR PC -> MAR PC -> MAR
MDR -> IR MDR -> IR DR -> IR
IR -> MAR IR -> MAR IR -> MAR
MDR -> A A -> MDR* A + MDR -> A
PC + 1 -> PC PC + 1 -> PC PC + 1 -> PC
